Guia de uso — VarMonitor

Que es VarMonitor?

Herramienta de monitorizacion en tiempo real de variables de una aplicacion C++. Permite visualizar valores, plotear graficos, configurar alarmas, generar funciones y grabar datos.

Columna 1 — Variables disponibles

  • Arbol de variables: lista jerarquica de todas las variables publicadas por la aplicacion. Se actualiza automaticamente cada 30s o con el boton ↻.
  • Filtro: escribe para buscar variables por nombre.
  • ▼ / ▶: colapsar/expandir todos los grupos.
  • Sel. todo / Ninguno: seleccionar o deseleccionar todas las variables visibles.
  • Monitorizar →: pasar las variables seleccionadas a la columna de monitorizacion.

Columna 2 — Monitorizando

  • Valor en directo: cada variable muestra su valor actual actualizado en tiempo real.
  • Doble click en el valor: editar y enviar un nuevo valor a la aplicacion (Enter confirma, Escape cancela).
  • Dropdown (—, G1, G2...): asignar la variable a un grafico. Selecciona + Nuevo para crear un grafico nuevo.
  • Click en el nombre: desplegar panel de detalles con min/max, alarmas y generador de funciones.
  • ● REC: grabar valores de las variables monitorizadas a un archivo TSV. Al parar se descarga automaticamente. Maximo 5 minutos por archivo.
  • Quitar sel.: eliminar variables asignadas a graficos de la lista de monitorizacion.
  • ⚠ Clear: eliminar todas las alarmas activas de golpe.
  • 💾 / 📂: exportar/importar la configuracion completa (variables, graficos, alarmas) a un archivo JSON.
  • Reset: limpiar toda la configuracion.

Panel de detalles (al hacer click en nombre)

  • Min / Max: estadisticas del historial reciente.
  • + Alarma: configurar umbrales Lo/Hi. Si el valor sale del rango, se activa una alarma visual y se genera un TSV automatico con los ultimos 10s de datos.
  • Generador: selecciona un tipo de funcion (seno, escalon, rampa, cuadrada, triangular, chirp, pulso, ruido), configura los parametros y pulsa ▶ para inyectar valores calculados a la variable. ■ Stop para detener.

Columna 3 — Graficos

  • Los graficos se crean al asignar variables desde el dropdown de la columna 2.
  • Si un grafico se queda sin variables asignadas, se elimina automaticamente.
  • × en la cabecera del grafico: eliminar ese grafico.
  • ▶ Play / ▮▮ Pausa: pausar o reanudar la actualizacion de todos los graficos.
  • 📷 PNG: capturar todos los graficos visibles como una sola imagen PNG.
  • Ventana: seleccionar el rango temporal visible (10s a 10min o todo el historial).
  • Buffer: memoria interna del navegador. Controla cuantos segundos de datos se acumulan en RAM (30s a 30min). Si el monitor consume mucha RAM, reduce este valor.
  • Crosshair: al pasar el raton, se muestra el valor de todas las trazas en ese instante.
  • Zoom: arrastra un rectangulo sobre el grafico para hacer zoom. Usa la rueda del raton para zoom rapido.
  • Pan: pulsa el boton de pan en la barra de herramientas y arrastra para desplazar la vista.
  • Autoscale / Reset: pulsa el boton de reset en la barra para volver a la vista completa.
  • ⚠ Icono de alarma: las variables con alarmas configuradas muestran un ⚠ junto al nombre en la columna 2, visible incluso colapsado.

Notificaciones de alarmas

  • Si el navegador no esta en primer plano cuando salta una alarma, se envia una notificacion del sistema.
  • Haz click en la notificacion para volver al monitor. Se requiere aceptar el permiso de notificaciones la primera vez.

Variables computadas

  • fx+ en la cabecera de Columna 2: crear una variable virtual calculada a partir de otras.
  • Escribe un nombre y una expresion JS (ej: sensors_temperature - control_pid_kp). Los puntos de los nombres se reemplazan por _.
  • Se evaluan en tiempo real y se pueden plotear como cualquier variable.

Cabecera

  • Host / Puerto: indican a que instancia de VarMonitor C++ se conecta el backend. Por defecto apunta al propio equipo donde corre el monitor web, pero puedes cambiar el host (otra IP accesible desde ese equipo) y el puerto (en el rango configurado, por defecto 1900–1910).
  • Poll (ms): intervalo de actualizacion de datos. Valores bajos = mas fluidez, mas carga.
  • Estado de conexion: indicador verde (conectado) o rojo (desconectado) del WebSocket.

Atajos de teclado

  • Espacio: pausar / reanudar graficos
  • R: iniciar / detener grabacion TSV
  • S: capturar screenshot de graficos
  • H o ?: abrir / cerrar esta ayuda
  • Escape: cerrar ayuda o descartar banner de alarma

Arquitectura y comunicacion

El sistema tiene tres piezas:

  1. Tu aplicacion C++ con libvarmonitor integrada: publica las variables y escucha conexiones TCP.
  2. Backend Python (FastAPI): se conecta por TCP a la app C++ y sirve esta pagina web.
  3. Este navegador: se conecta al backend Python via WebSocket para recibir datos en tiempo real.
  App C++          Backend Python         Navegador
 (tcp_port)  <---TCP--->  (web_port)  <---WebSocket--->  UI

Configuracion: varmon.conf

Toda la configuracion de puertos e IPs esta en un unico archivo de texto plano varmon.conf:

# VarMonitor configuration
tcp_port = 9100          # Puerto TCP (C++ escucha, Python conecta)
web_port = 8080          # Puerto de esta web (solo Python)
host = localhost          # IP de la app C++ (cambiar para acceso remoto)
history_capacity = 100    # Muestras por variable en buffer C++
  • tcp_port: puerto TCP donde la app C++ escucha. El backend Python se conecta a este puerto.
  • web_port: puerto donde se sirve esta pagina web.
  • host: IP o nombre del equipo donde corre la app C++. Usa localhost si todo esta en la misma maquina, o la IP del equipo remoto (ej. 192.168.4.1) para acceso por red.
  • history_capacity: numero de muestras que el C++ mantiene por variable en su buffer circular. Con sample_interval=100ms y capacity=100, el buffer cubre 10s. Valores bajos ahorran memoria con muchas variables.

Historial incremental

El monitor usa un sistema de entrega incremental de datos para optimizar el rendimiento:

  • El C++ muestrea las variables a su ritmo y las guarda en un buffer circular pequeno (history_capacity muestras).
  • El frontend pide al C++ solo los datos nuevos desde la ultima peticion (usando un numero de secuencia interno).
  • El frontend acumula su propia historia localmente para poder plotear ventanas de tiempo amplias (hasta 10 minutos).
  • Esto reduce drasticamente el trafico de red: solo se transmiten los puntos nuevos, no el buffer completo cada vez.

Acceso remoto

  • Para acceder desde otro PC en la misma red, abre http://<IP-del-servidor>:8080 en el navegador.
  • Si la app C++ esta en otra maquina, edita host en varmon.conf con la IP correspondiente.
  • Solo se transmiten por red las variables que estan siendo monitorizadas (no todas).

Cambiar la ruta de varmon.conf

  • Variable de entorno: VARMON_CONFIG=/ruta/al/archivo.conf
  • En C++: varmon::set_config_path("/ruta/...") antes de start()
  • Por defecto busca ./varmon.conf en el directorio de trabajo.
  • Si no existe el archivo, se usan los valores por defecto (9100, 8080, localhost).
